ConfigMap

ConfigMap — это объект Kubernetes, предназначенный для хранения конфигурационных данных в формате ключ-значение, которые могут использоваться Pod'ами и контейнерами.

ConfigMap позволяет отделить конфигурацию приложения от контейнерного образа.

Типичные данные в ConfigMap:
переменные окружения
конфигурационные файлы
параметры приложения
URL сервисов
feature flags

Когда использовать ConfigMap

Используйте ConfigMap для хранения:
конфигурации приложения
параметров окружения
конфигурационных файлов
настроек сервисов

Не используйте ConfigMap для:
паролей
токенов

ключей API

Для этих целей используется Secret.
